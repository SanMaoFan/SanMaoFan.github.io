echarts图表设置宽度100%结果为100px

发表于 2022-11-20 | 更新于 2023-06-13 | 分类于 前端

场景

在开发项目时，使用 element-ui 中的 Tabs 组件配合 ECharts 开发图表，在每一个 Tabs 标签页设置图表元素的宽度都为 100%，但只有最开始展示的标签页内的图表是 100% 宽度，而切换其他标签页时，其对应标签页下的图表元素宽度却是 100px

原因

element-ui 中的 Tabs 组件内的标签页是同时渲染的，通过 display: none; 样式来控制标签的切换，而 ECharts 在渲染图表时，获取不到父元素的宽度（也就是 Tabs 标签页元素宽度），所设置的图表 100% 宽度也就无效，ECharts 内部逻辑最后会返回 100px；

而在 echarts.js 文件中，渲染元素时，会获取元素的宽度，以下以 http://lib.baomitu.com/echarts/4.7.0/echarts.js 文件内的源码为例子，其内部获取元素宽度的方法为 _getSize，当 _getSize 方法参数传入 0 时，则是获取元素的宽度 width

方案一：v-if
