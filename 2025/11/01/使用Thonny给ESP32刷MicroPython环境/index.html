使用Thonny给ESP32刷MicroPython环境

发表于 2025-11-01 | 更新于 2025-11-02 | 分类于 IoT

准备环节

首先需要在电脑安装 python 环境，前往 python 官网 下载 python 环境并进行安装，推荐下载版本在 3.9.0 或以上。

以下为本次使用到的 ESP32 开发板，要准备的资料是以该板子为基础而收集的。

接着要找到关于开发板的芯片驱动，其芯片驱动主要用于 USB 转串口通信，这个需要根据开发板的具体规格来查找对应驱动，不然就算安装好驱动，将板子连接到电脑 USB 口后，却无法找到开发板的串口；本人的板子所使用的是 CH340 驱动，这是 CH340 下载地址

然后是要开发的 IDE 工具，这里推荐开源并较简单使用的 Thonny 软件，推荐下载 Thonny 4.0.2 版本，接着去往 MicroPython 官网下载所需要的 ESP32 固件，具体下载什么固件，要根据自己购买的板子规格来确定，本人所使用的固件是 ESP32 / WROOM，根据板子规格找到对应的 .bin 文件进行下载。

最后需要找一根板子插口对应的电脑 USE 连接线，用于开发板与电脑的连接，接下来便可以开始烧录开发板的环境。

安装 CH340 驱动

首先要做的是安装 CH340 驱动，其主要作用是当开发板连接上电脑 USB 接口后，能在设备管理器中看到开发板，打开下载好的 CH340 驱动程序，点击安装即可。

安装完毕后，使用数据线连接上开发板与电脑，接着可在电脑中的设备管理器 > 端口(COM 和 LPT) 中看到连接上的开发板。

如果查看不到已连接的开发板，需要再次确认是否安装了正确的驱动。

使用 Thonny 烧录固件

打开 Thonny 软件，点击右下角的环境，并点击 Configure interpreter...

然后从上往下依次选择解释器、开发板的串口，最后点击 install or update MicroPython。

进入 install or update MicroPython 窗口中，从上往下依次选择开发板串口、要烧录的固件的存储位置，最后点击 install

最后等待其固件烧录完成即可。

接着回到 Thonny 开始界面，右下角环境会自动切换到开发板的串口，并且控制台会进入刚刚烧录完的 MicroPython 环境。如果控制台未进入 MicroPython，那么是固件没有烧录成功，需要确认烧录的固件是否符合开发板硬件环境。

在控制台输入 help() 命令便可以进行交互。

至此，ESP32 开发板烧录 MicroPython 环境便完成了。
