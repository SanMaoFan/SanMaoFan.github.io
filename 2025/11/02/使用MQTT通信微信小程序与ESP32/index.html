项目介绍

本文章将使用 uni-app(vue3版本) 开发微信小程序，并通过 MQTT 技术与烧录了 MicroPython 环境的 ESP32 开发板进行通信，目的是实现微信小程序发布信息控制 ESP32 点亮开发板上的 LED，其中，MQTT 的代理服务由 EMQX Edge 工具提供，此次的项目将在局域网内进行。

准备工作

关于 EMQX Edge 工具的简单使用以及 ESP32 烧录固件 ，需要参照之前写的文章，MQTT 入门、使用Thonny给ESP32刷MicroPython环境

这里介绍使用 vsCode 开发 uni-app 的步骤，这里使用的 uni-app 项目是 vue3 + ts 的模版，如果不想要此模版，也可前往 uni-app 官网 查看其他模版创建方式，关于初始化 vue3 + ts 模版的 uni-app 项目，有以下几种创建方式供选择。

创建好项目后，用 vsCode 打开该项目，接着在 vsCode 控制台进行安装依赖包，例如使用 pnpm install，安装完依赖后，进入该项目的 src 目录，找到 manifest.json 文件，将这个文件中的 appid 设置为微信小程序的 AppID(小程序ID)，接着在 vsCode 控制台中输入启动微信小程序项目的命令，具体以项目中的 package.json 文件中的启动命令为准，例如我创建的项目启动微信小程序的命令为 pnpm dev:mp-weixin。接着便会在项目根目录中生成一个 dist 文件夹，这个文件夹中的 dev>mp-weixin 路径便是开发环境测试用的微信小程序项目，需要使用微信开发者工具去打开，如果没有微信小程序开发者工具，需前往 微信官网文档 进行下载。微信开发者工具导入开发环境生成的项目后便可以查看到项目内容，其项目结构与原生小程序项目结构很像，在 vsCode 工具中修改内容时，微信开发者工具会进行热更新，但如果其代码逻辑没有生效，则需要点击微信开发者工具的编译按钮进行刷新。因为此次项目是使用局域网进行测试的，所以需要更改微信开发者工具中关于请求域名的配置，点击工具右上角的详情按钮，进入本地设置选项卡，勾选下面的不校验合法域名、web-view(业务域名)、TLS版本以及 HTTPS 证书。

ESP32 接入 MQTT 技术

首先使用数据线连接 ESP32 开发板与电脑，并打开 Thonny 工具切换到 ESP32 开发板的串口。本次开发的 MicroPython 代码所要实现的功能是：ESP32 连接 WIFI 后再进行 MQTT 服务的连接，连接 MQTT 服务后订阅信息，并且根据订阅信息控制 ESP32 开发板上的 LED 灯亮灭。

关于本人手上的 ESP32 开发板中有一颗 LED 灯，其引脚为 D2，所以可以使用 Pin(2, Pin.OUT) 代码来定义该引脚，接着再控制此引脚的高低电平。

当烧录的程序初始化启动时，会发布一条信息到主题 testtopic，接着回到微信小程序，因为小程序端已经订阅了 testtopic 主题，所以刚刚 ESP32 所发送的信息会抵达小程序端。

接着点击小程序端的“发送信息”的按钮，会 Thonny 控制台中看到接接收到的消息。
