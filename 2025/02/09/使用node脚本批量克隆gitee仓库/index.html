使用node脚本批量克隆gitee仓库

发表于 2025-02-09 | 分类于 前端

场景

之前在 gitee 中，fork 了很多仓库，现在想都克隆到本地，但是一个个仓库进行克隆肯定耗时耗力，所以想到能否使用 node 脚本进行批量克隆仓库。

实现原理

在 gitee 中有提供 API 文档，可以使用 gitee.com/api/v5/user/repos 接口获取用户的所有仓库，但是前提必须有用户的私人令牌，然后在 node 脚本中，使用请求工具 utils-lib-js 来请求接口，并结合 child_process 启动一个子进程并输出命令显示是否克隆仓库成功。

获取 gitee 私人令牌

首先登录 gitee，然后点击右上角的头像下的 menu 中的”账号设置”或”设置”。

接着在新页面点击左侧的“私人令牌”选项，进行令牌的配置或是复制已有的令牌。

在创建私人令牌时，需要注意的是其权限有哪些，这里默认勾选全部权限，接着提交进行下一步。

接着 gitee 会提示账号安全验证。

最后即可拿到新生成的私人令牌。

测试接口

进入 Gitee API 文档，点击右上角的按钮进行测试。

接着会提示 Swagger 客户端对于私人权限的授权申请，这里以所有权限为例。

同意授权后，便进入到了测试页面，这里在左侧找到 仓库》列出授权用户的所有仓库 的接口，点击左侧下的”测试”按钮进行测试。

接着便可以在”测试”按钮的下方查看到返回的结果。

node 脚本代码

在 node 环境中，可以使用 http 或 axios 模块对数据进行请求，在此使用 utils-lib-js 来实现接口请求，接着使用 child_process 启动子进程来执行克隆命令，并输出是否克隆仓库成功。

最后在该项目中的控制台中执行该 js 脚本便可以批量克隆仓库在对应的文件夹中，在克隆所有仓库或者所有仓库对应执行代码完成后，其脚本程序会自动退出，程序自动退出，则表示脚本完全执行完毕，如果在脚本执行过程中手动退出，可能造成仓库克隆不全或者取消克隆的情况。

如果有克隆失败的仓库，需要需要查看对应的错误提示什么，在进行对应的代码或者配置改动。
