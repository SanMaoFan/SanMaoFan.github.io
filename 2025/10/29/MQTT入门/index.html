什么是 MQTT

MQTT 称为消息队列遥测传输，是基于发布订阅模式的消息协议，是为硬件性能低下的远程设备以及网络状况糟糕的情况下而设计的，MQTT 与 HTTP 一样是个网络协议，因为轻量简单并且很多时候传输效率是后者的数十倍，并且有 QoS 支持，数据得到了可靠的传输保证，所以后续成为了物联网的最佳选择，在日常中，MQTT 的使用场景特别广泛，例如使用共享单车时，就用到了该协议。

QoS 分为三种级别：级别0、级别1、级别2。级别0表示最多发一次消息，发送者会想尽办法发送消息，不保证消息的传递一定完成，当客户端以 QoS 0 级别发布信息时，MQTT 代理不确认收到，并且消息不被发布者存储和重新发送，这个级别通常称为“即发即弃”，提供与底层 TCP 协议相同的保证；级别1表示至少发一次，QoS 1 保证一条信息至少一次传递给接收方，如果消息接收者没有收到消息或者消息本身丢失，消息发送者会再次发送，以保证消息接收者至少会收到一次信息，但这样可能会多次传递消息，使得消息重复；级别2表示只有一次交付，通过确认机制确保消息仅送达一次，这是最高等级的服务质量，消息丢失和重复都是不可接受的，但也带来了最复杂的交互流程和最高的开销，每一次的 QoS 2 消息投递，都要求发送方与接收方进行至少两次请求/响应流程。

MQTT 中有三个重要角色：代理(Broker)、发布者(Publisher)、订阅者(Subscriber)。这里以小破站(哔哩哔哩)为例，小破站作为代理方，每一个 UP 主都是发布者，当一个观众关注了喜欢的 UP 主，那么观众便成了订阅者，当 UP 发布了新视频，会通过小破站代理方通知到订阅方(观众)。

使用 EMQX Edge 与 MQTTX 展示例子

使用 EMQX Edge 当代理，并配置发布、订阅模块

首先下载 EMQX Edge 软件，这是为了做本地的自托管服务，也就是作为上面所说到的 Broker 代理方。

下载该压缩包后，解压并进入解压后的文件夹根目录，可以看到一个 nanomq.exe 文件，使用 cmd 进入该根目录路径，并输入命令 nanomq start 或者官方推荐的 ./nanomq start 命令，接着便会启动该软件，并最后提示软件服务开启成功，并且会有一个 HTTP 服务，该地址可以在浏览器打开。

浏览器输入 localhost:8081 即可打开服务，登录时的默认账号为: admin，默认密码为: public。

接着进入工具界面进行配置连接、发布者、订阅的信息。

如果连接成功，会提示该客户端已连接。

接下来便是设置 Subscription(订阅) 以及 Publish(发布) 模块，在 Subscription(订阅) 模块中，需要定义 Topic(主题)，Topic(主题) 的作用类似于是给订阅者与发布者唯一房间号，这样两者才会有碰面(收发信息)的机会，这里将 Topic(主题) 设置为 testtopic，是指本服务收取 testtopic 主题发布的信息，而 QoS 则默认设置 0，点击订阅按钮后即可订阅该主题的信息，那些没有被订阅的主题，其主题无论怎么发布信息，本服务是收取不到任何内容的。

在 Publish(发布) 模块中，主要作用是发布信息给什么主题，以及收取已订阅主题的信息。其中在 Received(接收) 小模块中，不是什么信息都会接收，只会接收刚刚在 Subscription(订阅)模块中已订阅的主题的信息。例如，当前在 Publish(发布) 模块下方的 Topic(主题)输入框中填入刚刚订阅的 testtopic 主题，并点击发布按钮，则可以看到最新发布的信息被接收到了。

为了不混淆信息，先将 Publish(发布) 模块下的主题输入框修改为 testtopic1

使用 MQTTX 接收和发布信息

首先下载 MQTTX 软件，安装该软件后需要先连接上刚刚启动的代理服务，先点击新增连接进入连接的配置页面。

配置完后点击右上角的连接，就可以测试连接已启动的服务，接下来便可以看到以下面板，这是面板的大致介绍。

这里比较重要的是接收信息的主题，在托管服务端中，已经订阅了 testtopic 主题，那么MQTTX 软件端想要发送消息到托管服务端，就需要设置相同的主题，也就是前面比喻的唯一房间号，那么接下来点击 MQTTX 软件右下角绿色小飞机图标就可以发布信息到 testtopic 主题了，这时候切换到托管服务端，就可以看到 MQTTX 发送来的信息。

接下来是 MQTTX 软件新增订阅，托管服务端的 Publish(发布) 模块下有一个主题 Topic 输入框可以填写，上面说过这是填写什么主题，其发布的信息就会去往哪个主题内，那么在 MQTTX 中添加一个订阅，设置为 testtopic1，以便接受服务端准备发布的信息。

MQTTX 软件新增订阅完毕后，在托管服务端中发布一条信息，在 MQTTX 软件中会显示该信息来自哪个主题，包含什么内容。
