wxLogin--getUserInfo

发表于 2019-12-24 | 更新于 2020-01-05 | 分类于 前端

公司有一个项目是关于后台管理相关设备数据的，其中的登录不是以往的账号密码登录获取用户相关信息，而是采用微信扫码登录，然后再获取用户相关信息，这样做的考虑是：本地用户信息过期了，不用回到登录页面重新登录，只需扫描页面中弹出的二维码即可继续使用管理后台。

先简单介绍下wxLogin.js的使用方法:

本次项目是下载到本地引入使用，查看源码中后，可知二维码是嵌套在一个iframe中的

效果可以像网页登录的效果一样

在项目中，是用两个页面来作为重定向路径，一个login页面是用户第一次进入管理后台时，所展示的，并且也只展示这一次，因为后面用户信息超时而需要重新扫码时，所重定向的页面就为code页面。

login页面很简单，就是获取后台相关信息，然后展示二维码给用户扫描，用户也就可以操作管理后台了；当用户信息过期后，会新建一个id为newCode的元素，展示效果以及具体配置如下：

newCode元素内部就为了展示二维码，用户扫描登录后，就进入到code页面(是在iframe层中显示code页面)，而code页面中是使用了 self.location.href != top.location.href 是否不相等的方法来跳出iframe层，并且将newCode元素去除，这样就可以使浏览器更新了用户信息，也避免了iframe层中打开页面的情况。
