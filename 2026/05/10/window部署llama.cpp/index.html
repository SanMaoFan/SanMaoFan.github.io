window部署llama.cpp

发表于 2026-05-10 | 更新于 2026-06-17 | 分类于 前端

下载 Windows 环境的 llama.cpp

前往 llama.cpp 官网地址：https://llama-cpp.com/download/ 并进入 window 环境进行下载。

这里是跳转到 github 的地址，会有几种类型提供下载————CPU、CUDA、Vulkan 等类型，这里需要选择合适的类型，若没有独立显卡，推荐选择 CPU 类型，由于本机有 NVIDIA 独立显卡，并且安装了 CUDA 12 的驱动，所以选择 Windows x64 (CUDA 12) 类型进行演示，推荐选择对应的显卡驱动版本，这样能更好的发挥显卡性能。

将下载的程序添加到环境变量

在下载完毕后，将文件解压到自己对应的硬盘分区中，例如本机解压路径为： D:\soft\installed\llama-b9644-bin-win-cuda-12.4-x64，需要记住此路径，然后打开 window 的环境变量，打开用户变量中的 Path 变量，将刚刚的解压路径添加进去并保存；

接着打开任意命令行窗口，输入 llama-cli 命令，如果不是提示 “llama-cli 不是内部命令”，而是提示 error: --model is required就说明环境变量已设置完成；

下载模型

接下来是需要下载 AI 模型，llama.cpp 是需要加载模型才能运行的，国内直接访问 Hugging Face 的速度会较慢，推荐使用国内的镜像站：https://hf-mirror.com/，这里以 Qwen3.5-9B 版本的模型作为示例，具体版本链接为： https://hf-mirror.com/unsloth/Qwen3.5-9B-GGUF/tree/main，接着选择 Qwen3.5-9B-Q4_K_M.gguf 模型，具体模型应该根据电脑性能来选择，量力而行；

测试模型

模型文件下载完毕后，将模型解压到 llama.cpp 程序已解压的文件夹内，例如上文示例的：D:\soft\installed\llama-b9644-bin-win-cuda-12.4-x64，接着在 llama.cpp 所在文件夹中打开命令行窗口，输入命令：llama-cli -m Qwen3.5-9B-Q4_K_M.gguf，然后等模型运行起来即可，第一次运行会有点慢，需要耐心等等，启动完成后，便可以在命令行窗口中对模型进行操作；

如果不喜欢命令行窗口的形式，llama.cpp 还有 webUI 页面，使用命令 llama-server -m Qwen3.5-9B-Q4_K_M.gguf 启动模型即可，默认在浏览器打开 http://127.0.0.1:8080 即可看到页面；
