js判断对象自身为空的方案

发表于 2023-12-24 | 更新于 2023-12-25 | 分类于 前端

JSON.stringify

JSON.stringify 方案可以使对象序列化，转为相应的 JSON 格式。

但该方案有缺点，如果该对象中有属性的值为 undefined、任意函数以及 symbol 值，在序列化过程中会被忽略掉或是转为 null 出现在数组中（属性的值为数组时，如果其内容包含 undefined，在转化时会被转为 null）

for in 搭配 hasOwnProperty

使用 for in 可以对对象进行遍历，并且可以遍历到对象的原型链上的属性。

如果不想遍历到对象原型链上的属性，可以通过 hasOwnProperty 方法来避开原型链上的遍历。

可以使用 hasOwn 方法替代 hasOwnProperty，具体用法是将以上代码中的 obj.hasOwnProperty(i) 替换为 Object.hasOwn(obj, i)；值得注意的是 for in 方法 不可以遍历到对象上的 symbol 和不可枚举的属性，所以还需要找更好的替代方案。

Object.keys

Object.keys 会返回对象可枚举属性组成的数组，并且不会遍历原型链上的属性。

但 Object.keys 和 for in 一样，都只能遍历可枚举属性，如果使用 Object.defineProperty 将属性设置为不可枚举的话，那么该属性便会被略过。

Object.getOwnPropertyNames

使用 Object.getOwnPropertyNames 可以得到对象自身的所有属性名组成的数组（包括不可枚举的属性）。

但该方法不能获取 symbol 值作为名称的属性，以上介绍的 JSON.stringify 、for in 以及 Object.keys 方法也不能获取到 symbol 值作为名称的属性。

Object.getOwnPropertyNames 结合 Object.getOwnPropertySymbols

已知 Object.getOwnPropertyNames 唯一的缺点是不能获取 symbol 的值作为名称的属性，而 Object.getOwnPropertySymbols 只能获取有 symbol 值作为名称的属性，于是将两者结合一起使用便能解决该缺点。

经过以上测试，证明方法有效，但是判断方式有点繁琐，于是引出下一个方案

Reflect.ownKeys

Reflect.ownKeys 方法返回一个由目标对象自身的属性组成的数组，它返回的值相当于 Object.getOwnPropertyNames(target).concat(Object.getOwnPropertySymbols(target))。

所以，判断一个对象是否为空时，如果浏览器支持，那么使用 Reflect.ownKeys 方案是最好的。
