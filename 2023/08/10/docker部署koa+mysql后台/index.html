服务器使用 docker 安装 mysql 并创建数据库

首先，需要在服务器上安装 mysql，这里使用 docker 来安装，使用 docker 来拉取已有的镜像，这里使用 mysql 5.7 版本为例子，拉取 mysql 后，启动该容器并进入容器，以下为在服务器控制台中依次输入命令的顺序：

在启动容器后，可以先使用 docker ps -a 命令查看容器是否已经启动，如果有关于 mysql 的容器，并状态不为 Exited，则表示容器启动成功

在进入容器后，再输入 mysql -u root -p 便可进入 mysql 登录的操作，输入登录密码后便进入到了 mysql，接下来按照在本地 mysql 创建数据库、表、表数据的做法来创建数据。

制作项目镜像并启动容器

注意！上传项目前，最好将 mysql 配置中的 host 属性设置为线上服务器的 ip

注意！上传项目前，最好将 mysql 配置中的 host 属性设置为线上服务器的 ip

将后端项目上传至服务器（不必上传 node_module 文件夹），例如上传到一个叫 backend 的文件夹，然后进入到 backend 文件夹内，在控制台输入 docker build -t yun-work-backend:v1 . 命令，项目便会进行初始化并将项目制作成镜像。

在制作完镜像后，并可以启动容器，输入命令 docker run -d --name yunbackend -p 8888:8844 yun-work-backend:v1 便可以启动容器，这里将项目的服务挂载在 8888 端口，后续在请求时，需要请求此端口才会有数据，使用 docker ps -a 命令便可以查看该容器是否已启动

如果该容器状态为 Exited，那么可以通过 docker logs yunbackend 来查看该容器的打印日志，如果一切正常，那么使用 docker logs 命令可以看到控制台输入了项目启动后的日志

最后，使用接口测试工具来向线上请求接口，记得将端口号改为 docker 放出的端口号

如果请求失败，应查看后端项目容器 yunbackend 里打印的日志，如果提示没连接上数据库或者连接失败，那么使用命令 docker exec -it mysql bash 进入容器，登录 mysql 查看是否正常；需要注意的是，所有涉及端口的步骤，都需要让服务器开发其端口！例如3306、8888
