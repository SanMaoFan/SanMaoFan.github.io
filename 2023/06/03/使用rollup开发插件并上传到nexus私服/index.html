场景

在日常开发中，项目需要使用各式各样的插件来辅助工作，一般是从 npm 上下载即可，但如果是公司内部需要在特定场景使用自己研发的工具，并且只供内部下载使用，那么便需要从零开发称手的插件

准备

开发插件的工具这里挑选 rollup 工具（rollup 中文网），而放置插件的仓库则是使用 nexus。

rollup 部分

rollup 介绍

rollup 是一款 ES module 的打包器，可以将项目中的细小模块打包成整块的代码，使得代码可以更好的运行在浏览器环境或者是 nodejs 环境，虽然 rollup 与 webpack 非常类似，但 rollup 要小巧的多，因为 rollup 仅仅是一款 ESM 的打包器，没有其他额外的功能，例如 rollup 不支持类似 HMR（热更新）这种高级特性

rollup 可以安装至全局(npm install rollup -g)，或者作为项目的开发依赖去安装(npm install rollup -D)，本文以作为项目开发依赖的情况为例子，并默认使用 typescript 语法开发插件。

nexus

nexus 是做为 Maven 的仓库管理器，它极大地简化了自己内部仓库的维护和外部仓库的访问。其仓库也可以放置前端需要使用到的 js 插件，关于部署以及配置可参考这篇文章

上传打包后的插件到私服

首先要进入到项目中 package.json 文件设置入口文件以及要上传的内容，在 package.json 文件中的 main 属性声明项目的入口文件，本项目中 main 属性的值为 dist/index.ts，因为要上传的是 dist 文件夹，所以主要入口便是根据 dist 开头，而要指定上传什么文件，则是在 files 属性中设置，这里设置为 ["dist"]，便可只上传 dist 文件夹上私服，除了这两个属相以外，还可以设置 module 以及 browser 属性，这两个属性，和 main 属性的作用相同，都是设置引入时所采用哪个文件，而 main、module、browser 这三个属性只要是为了在用户 import 私服的插件时，其查找规则会按 browser+mjs > module > browser+cjs > main 的顺序来查找。

如何从私服下载插件

将插件上传到私服后，新项目中该如何下载使用呢？一般下载插件时，只需要在控制台输入 npm i 或者 cnpm i 便可以下载项目所需要的所有插件，但是这种下载是向特定的源（镜像）上请求的，例如：https://npmmirror.com/mirrors，或是其他切换后的源，但是公司内部的插件是存在公司私服内的，那么如何在下载过程中，既能从第三方下载所需插件，又能下载私服的插件呢？

什么是 .npmrc 文件？

.npmrc，可以理解成npm running cnfiguration, 即 npm 运行时配置文件。简单点说， .npmrc 可以设置 package.json 中依赖包的安装来源，当输入 npm install 时，项目会自动根据配置文件判断该从什么地方下载什么依赖包。

多个 .npmrc 文件时的优先级

如果电脑中有多个 .npmrc 文件，当我们安装插件时，npm 按照如下顺序读取这些配置文件：

  • 项目配置文件: /project/.npmrc
  • 用户配置文件：~/.npmrc
  • 全局配置文件：$PREFIX/etc/npmrc
  • npm 内置配置文件 /path/to/npm/npmrc
