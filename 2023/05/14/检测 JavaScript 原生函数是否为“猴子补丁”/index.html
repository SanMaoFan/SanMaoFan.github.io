检测 JavaScript 原生函数是否为“猴子补丁”

发表于 2023-05-14 | 更新于 2023-06-05 | 分类于 前端

JavaScript 中的原生函数

在 JavaScript 中，“原生函数”是指源代码被编译为原生机器码的函数。例如 JavaScript 中标准内置对象的原生函数，诸如 eval()，parseInt() 等，又或者在浏览器 Web API 中找到 fetch()，localStorage.getItem() 等。

由于 JavaScript 的动态特性，开发者可以覆盖浏览器暴露出的原生函数。这种技巧被成为 猴子补丁(Monkey patching)

什么是”猴子补丁”

猴子补丁主要用于修改浏览器内置 API 和原生函数的默认行为。这通常是添加特定功能、polyfill 特性、hook 到 API 的唯一方法，因为没法直接对这些 API 进行访问。

例如 Bugsnag 这样的监测工具，重写了 Fetch 和 XMLHttpRequest 的 API 来获取由 JavaScript 代码触发的网络连接相关信息。

猴子补丁即强大又危险，被覆盖的代码会脱离控制：未来 js 引擎的更新可能会打破你在补丁中做出的一些设定，并导致严重的 bug。另外，在引入由他人开发的猴子补丁时，可能造成一些已开发的猴子补丁之间的覆盖，从而造成严重的 bug。

由于种种原因，有时需要鉴定给定函数是否为原生函数，即是否被打了猴子补丁。

toString() 检测法

检查一个函数是否“干净”（即没有猴子补丁）最常用的方法就是检查函数的 toString() 输出。

以 fetch 方法为例，其原生函数的 toString() 会返回一行 function fetch() { [native code] }

依照运行 JavaScript 引擎的不同，输出结果会有略有不同。但在大多数浏览器内，还是可以很安全的家丁返回的字符串会包含 “[native code]”。

  • iframe 有时会由于强 CSP 或者代码没有通过浏览器运行而导致不可用；
  • 如果第三方可以给 iframe API 打上猴子补丁，那也就无法 100% 信任 iframe 生成的 window 对象；
  • 修改或者调用 DOM 的原生函数（比如 document.createElement），导致没法使用这种方法，因为它们会指向 iframe 的 DOM 而不是顶层的 DOM
