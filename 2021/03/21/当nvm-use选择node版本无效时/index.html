当nvm-use选择node版本无效时

发表于 2021-03-21 | 分类于 前端

场景

因为换了电脑，所以需要将之前的hexo转移到新电脑，并且上传到gitee上，当所有步骤准备完毕时，输入命令行：hexo g -d。但命令行窗口中却提示错误：TypeError [ERR_INVALID_ARG_TYPE]: The "mode" argument must be integer. Received an instance of Object

nvm use切换版本无效

新电脑上安装的node版本为14.15.4，网上说是环境安装的node版本过高的问题，所以安装了nvm以及10.16.0版本的node。

但是在使用nvm use 10.16.0切换node版本，再使用命令行：hexo clean && hexo g -d，命令行窗口依旧报:TypeError [ERR_INVALID_ARG_TYPE]: The "mode" argument must be integer. Received an instance of Object，在使用nvm list查看node版本时，发现当前使用的node版本依旧为14.15.4，再次尝试切换node版本后，发觉nvm use命令行无效。

解决方案

在github上的nvm-windows的issues中可以找到解决方法，将node安装目录重命名：Renamed “C:\Program Files\nodejs” to “C:\Program Files\nodejsx”
