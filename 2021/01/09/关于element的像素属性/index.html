关于element的像素属性

发表于 2021-01-09 | 更新于 2023-06-05 | 分类于 前端

Element.client相关属性

以下client属性都是只读属性

Element.clientWidth

内联元素以及没有CSS样式的元素的clientWidth属性值为0。Element.clientWidth属性表示元素的内部宽度，以像素计。该属性包括内边距padding，但不包括边框border、外边距margin和垂直滚动条。可以通过CSS width + CSS padding - 垂直滚动条宽度计算得出。

当在根元素(元素)上使用clientWidth时(或者在上，如果文档是在quirks（怪异）模式下)，将返回viewport的宽度（不包括任何滚动条）。

Element.clientWidth的值会被四舍五入为一个整数。如果需要一个小数值，可使用element.getBoundingClientRect()。但需要注意的是，getBoundingClientRect()方法只能获取元素的width/height，但是获取的值是和offsetWidth/offsetHeight一样，包括边框的长度，所以不能获取clientWidth/clientHeight。

Element.clientHeight

对于没有定义CSS或者内联布局盒子的元素，返回值为0。clientHeight是指元素内部的高度，该属性包括内边距padding，但不包括边框border、外边距margin和水平滚动条。可以通过CSS height + CSS padding - 水平滚动条高度得出。

Element.clientWidth的值会被四舍五入为一个整数。如果需要一个小数值，可使用element.getBoundingClientRect()。但需要注意的是，getBoundingClientRect()方法只能获取元素的width/height，但是获取的值是和offsetWidth/offsetHeight一样，包括边框的长度，所以不能获取clientWidth/clientHeight。

Element.clientTop

一个元素顶部边框的宽度（以像素表示）。不包括顶部外边距和内边距。

Element.clientLeft

一个元素左边边框的宽度（以像素表示）。如果元素的文本方向是从右向左，并且由于内容溢出导致左边出现了一个垂直滚动条，则该属性包括滚动条的宽度。该属性不包括左边外边距和内边距。

HTMLElement.offset相关属性

以下offset属性都是只读属性

HTMLElement.offsetParent

返回一个指向最近的（指包含层级上的最近）包含该元素的定位元素或者最近的table,td,th,body元素。当元素的style.display设置为”none”时，offsetParent返回null。offsetParent很有用，因为offsetTop和offsetLeft都是要相对于其内边距边界来参考的。

浏览器兼容性：在webkit中，如果元素为隐藏的（该元素的或其祖先的style.display为”none”），或者该元素的style.position被设为”fixed”，则该属性返回null。

在IE9中，如果该元素的style.position被设为”fixed”，则该属性返回null。（display:none无影响）

HTMLElement.offsetLeft

对于块状元素，该属性返回当前元素border外侧左上角相对与HTMLElement.offsetParant节点border内侧的左边界偏移的像素值，和scroll无关。

如果是内联元素，offsetLeft和块状元素情况一样，依旧时border外侧到offsetParent节点border内测的距离；但是当其父元素使用了text-indent属性并且生效，内联元素在文字过多时会换行，这个时候会看到第一行文字和第二行文字左边的距离不同，使用offsetLeft时，采用的是第一行的左距离像素值，而不是第二行的左距离。

HTMLElement.offsetTop

它返回当前元素border外侧相对于其offsetParent元素border内侧的顶部的内边距距离。

HTMLElement.offsetHeight

该属性返回元素的实际（可见）像素高度，包含内边距、边框和水平滚动条的宽度，返回值为整数。如果元素被隐藏（例如元素或者元素祖先之一的元素的style.display被设置为none），则返回0。

HTMLElement.offsetWidth

该属性返回元素的实际（可见）像素宽度，包含内边距、边框和垂直滚动条的宽度，返回值为整数。如果元素被隐藏（例如元素或者元素祖先之一的元素的style.display被设置为none），则返回0。

Element.scroll相关属性

ScrollToOptions字典

ScrollToOptions字典当中的属性用于指定一个元素应该滚动到哪里，以及滚动是否应该平滑。

一个ScrollToOptions字典可以作为参数提供给以下的方法：

  • Window.scroll()
  • Window.scrollBy()
  • Window.scrollTo()
  • Element.scroll()
  • Element.scrollBy()
  • Element.scrollTo()

属性含义：

  • ScrollToOptions.top: 指定window或元素Y轴方向滚动的像素数
  • ScrollToOptions.left: 指定window或元素X轴方向滚动的像素数
  • ScrollToOptions.behavior: 指定滚动是否应该平滑进行，还是立即跳到指定位置。可选的值有smooth(平滑滚动)，instant(瞬间滚动)，默认值为auto（等同于instant）

Element.scrollTo()

该方法可以元素内部滚动到的指定坐标位置。可采用一次传入x,y的值，或者是传入一个ScrollToOptions的字典来让元素内部滚动。

Element.scrollBy()

该方法是使得元素内部滚动一段特定的距离。可采用一次传入x,y的值，或者是传入一个ScrollToOptions的字典来让页面滚动。

Element.scrollTop

该属性可以获取或设置一个元素的内容垂直滚动的像素数值。也就是获取或者控制元素的垂直滚动条的数值，数值是一个整数，但在显示比例缩放的时候，该数值可能为浮点数。

scrollTop可以被设置为任何整数值，同时注意：

  • 如果一个元素不能被滚动（例如，它没有溢出，或者这个元素有一个”non-scrollable”是属性），scrollTop将被设置为0.
  • 设置scrollTop的值小于0，scrollTop被设为0.
  • 如果设置了超出了容器可滚动的值，scrollTop会被设为最大值。

Element.scrollLeft

该属性可以获取或设置一个元素的内容水平滚动的像素数值。也就是获取或者控制元素的水平滚动条的数值，数值是一个整数，但在显示比例缩放的时候，该数值可能为浮点数。

需要注意的是，如果这个元素的内容排列方向(direction)是rtl(right-to-left),那么滚动条会位于最右侧，并且scrollLeft值为0.此时，当从右到左拖动滚动条是，scrollLeft会从0变为负数。如果scrollLeft的值不为0，那么从右往左拖动时，其数值是从最大值逐渐变小，最小为0。

scrollLeft可以是任意整数，然而：

  • 如果元素不能滚动（比如：元素没有溢出），那么scrollLeft的值为0。
  • 如果给scrollLeft设置的值小于0，那么scrollLeft的值将变为0。
  • 如果给scrollLeft设置的值大于元素内容最大宽度，那么scrollLeft的值将被设为元素最大宽度。

Element.scrollHeight

该属性为只读性质，代表元素的内容高度，包括由于overflow溢出而在屏幕上不见的内容区域。高度包含元素的内边距，不包含边框、外边距和垂直滚动条。该属性还可以包含伪元素的高度，例如::before和::after。如果元素的内容可以适合而不需要垂直滚动条，则scrollHeight等于clientHeight。

Element.scrollWidth

该属性为只读性质，代表元素的内容宽度，包括由于overflow溢出而在屏幕上不见的内容区域。宽度包含元素的内边距，不包含边框、外边距和垂直滚动条。该属性还可以包含伪元素的宽度，例如::before和::after。如果元素的内容可以适合而不需要水平滚动条，则scrollWidth等于clientWidth。

MDN中说，实际测试的过程中，谷歌获取的Element.scrollWidth和IE、火狐下获取的Element.scrollWidth的值并不相同。

window.scroll相关属性

其scroll()、scrollBy()、scrollTo()方法是和Element中的scroll()、scrollBy()、scrollTo()的使用方法对应，只不过作用对象变成了window而已。

window.scrollX & window.scrollY

window.scrollY: 该属性返回文档/页面垂直方向滚动的像素值。
window.scrollX: 该属性返回文档/页面水平方向滚动的像素值。

pageXOffset属性是scrollX属性的别名，pageYOffset属性是scrollY属性的别名；这些属性之间总是相等的，当为了浏览器兼容性时，在这些属性中，应当使用window.pageXOffset代替window.scrollX、window.pageYOffset替代window.scrollY。但是遇到旧版本的ie（<9）时，应当使用其他方案来解决兼容性问题（也就是使用scrollLeft和scrollTop属性替代）。

window.page相关属性

window.pageXOffset & window.pageYOffset

这两个属性为只读性质，并且在ie 8及更早ie版本不支持这些属性，但可以使用document.documentElement.scrollLeft和docuemnt.documentElement.scrollTop属性来替代
这连个属性返回文档在窗口左上角水平和垂直方向滚动的像素值。这两个属性相分别等于scrollX和scrollY属性。
