跟随window系统主题变换项目主题样式

发表于 2024-03-05 | 更新于 2024-03-07 | 分类于 前端

场景

在 PC 端，为了提高用户体验，项目中一般会预制 白昼/暗黑 主题供用户选择，可该主题方案是“固定”形式的，如果用户在系统中设置系统主题，那么项目的主题一直是项目中选定的主题，彼此之间没有什么联动，可以将项目主题与系统主题理解为两个瓶子里的水，一个瓶子里加入了什么颜料，都不会去影响另一个瓶子里水的颜色。但为了更好的用户体验，所以加入随系统主题切换项目主题的功能也应该要考虑到；项目中可通过原生 css 自定义属性(变量)、sass/less 等预处理器、动态加载link切换类名等方案来实现主题的切换，这里以原生 css 自定义属性的方案来实现此功能。

关于 prefers-color-scheme

prefers-color-scheme 是 css 媒体特性，用于检测用户是否有将系统主题色设置为亮色或者暗色。

语法

  • no-preference: 表示系统未得知用户在这方面的选项。在布尔值上下文中，其执行结果为 false。
  • light：表示用户已告知系统他们选择使用浅色主题的界面。
  • dark：表示用户已告知系统他们选择使用暗色主题的界面。

备注： “未得知”、“已告知”等用语，英文原文如此。 “未得知”可理解为：浏览器的宿主系统不支持设置主题色，或者支持主题色并默认为/被设为了未设置/无偏好。 “已告知”为：浏览器的宿主系统支持设置主题色，且被设置为了亮色或者暗色。

目前，若结果为 no-preference，无法通过此媒体特性获知宿主系统是否支持设置主题色，或者用户是否主动将其设置为无偏好。出于隐私保护等方面的考虑，用户或用户代理也可能在一些情况下在浏览器内部将其设置为 no-preference。

关于 Window.matchMedia()

Window.matchMedia() 用于得知指定的媒体查询字符串解析出的结果，此方法中会返回一个 matches 属性，此属性表示当前 Document 与媒体查询列表是否匹配（需要注意的是，这里更明确的说法是：当前 Document 是否适用于某种媒体查询，例如系统主题设置为暗色，而样式中并没有设置暗色的媒体查询，依旧会返回 true），返回结果为布尔值。
