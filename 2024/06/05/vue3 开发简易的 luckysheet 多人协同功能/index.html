了解 luckysheet 的数据格式

一般开发中，如果要展示 luckysheet 已有的内容，那么是向后端请求数据然后呈现出来，这里需要先了解 luckysheet 的数据格式，才知道接口该返回什么样的内容格式。

在 luckysheet 官方文档中，提供了 getAllSheets 方法获取所有的工作表配置；

在浏览器中可以看到 luckysheet 所有工作表的数据格式如下：

其数据格式表明，有几张表（sheet），就有几个对象放在数组之中，而每张表的数据放在该对象中的 data 属性之中，而 data 属性是一个二维数组，其 data 属性的索引代表行，而每一行数据拥有着一行所有列的单元格的数据，例如， data[1][2] 代表当前 sheet 表中的第二行第三列的单元格数据。每一个单元格拥有这个一个对象，代表该单元格所拥有的值、样式等数据，其中 v 属性代表当前单元格的原始值。

监听单元格编辑的方案

luckysheet 中提供了许多钩子函数，要做到多人协同，就需要监听用户改动单元格的操作，luckysheet 中提供了 cellUpdated 函数，在更更新单元格后便会触发

在浏览器中，改动第一行第一列的单元格，在控制台中可以看到对应的数据打印：

其五个参数分别表示：

  • row: 单元格所在行数
  • col: 单元格所在列数
  • before: 修改前的单元格对象
  • after: 修改前的单元格对象
  • isRefresh: 是否刷新整个表格

由此可以知道当前用户所在编辑的是哪行哪咧，并且输入的值是什么，但还有一个问题点：用户编辑的是哪张工作表？luckysheet 中提供了 workbookCreateAfter 以及 sheetActivate 钩子函数，分别代表的含义为：获取表格创建之后整个工作簿的配置，以及激活工作表前判断当前工作表的 index 属性、是否切换到了数据透视表页、是否新建了sheet页。首先在创建 luckysheet 时，可能是向服务器获取的数据，所以需要在 luckysheet 创建完毕后得知当前整个工作薄的配置，如果还是使用 window.luckysheet.getAllSheets 方法获取数据，则是不正确的数据，而 sheetActivate 函数则可以在用户切换工作表时，获取到用户当前要操作的工作表的 index 数据。

判断更新值以及填入更新值至单元格

首先是判断当前单元格的值和要更新单元格的值是否一直，如果不一致才更新，一致则退出更新，这里使用到的钩子函数为 cellUpdateBefore，其函数含义为：更新这个单元格之前触发，return false 则不执行后续的更新。在编辑状态下修改了单元格之后，退出编辑模式并进行数据更新之前会触发这个钩子。
该函数拥有四个参数，分别为：

  • row: 单元格所在行数
  • col：单元格所在列数
  • value: 要修改的单元格内容
  • isRefresh: 是否要刷新整个表格

设置好钩子函数后，就需要对比要修改的值是否相等，这里为了可以更新到值、样式等内容，所以对要更新的值做一个详细比较；并且更新值分为两种情况：1、本地更新单元格内容。2、ws 更新单元格内容。

最后，在浏览器中打开多个页面，对其中一个 Excel 进行编辑时，便会将另一个 Excel 相同的单元格一并更新
