微信小程序中 input 浮动在页面底部并聚焦时，页面会上移的问题

发表于 2024-11-01 | 更新于 2024-11-03 | 分类于 前端

场景

在使用 React + Tarojs 开发微信小程序项目时，遇到一个需求，开发一个咨询页面，将输入框 input 浮动在页面底部，发送的内容显示在页面上，但当底部的 input 输入框聚焦并弹起键盘时，其页面会上移，导致用户体验不好。

方案

首先，input 是靠浮动在页面 container 中的，container 原始高度设置为 100vh，input 需设置属性 adjust-position={false}。接着，需要监听 input 的 onFocus 以及 onBlur 事件，在 onFocus 事件中，其第二个 event 参数可以根据 event.detail.height 来获取键盘高度，这里的高度的单位是 px，这个高度是为了让页面 container 可以设置动态高度，从而让页面不会在键盘弹起时上移，而监听 onBlur 事件，是因为 input 失焦后键盘会收起，方便把 container 重新恢复到原来的高度。代码大概如下:
